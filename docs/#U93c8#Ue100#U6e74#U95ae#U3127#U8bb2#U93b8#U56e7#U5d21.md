# 背单词 App - Windows 本地部署指南

> 全程无需阿里云，所有服务跑在你的 Windows 11 本机上。

---

## 总览：你需要做的事

```
第 1 步  安装 PostgreSQL           → 约 5 分钟
第 2 步  把项目复制到 D:\wordbook_app  → 1 分钟
第 3 步  双击 setup.bat 一键初始化     → 约 3 分钟
第 4 步  双击 start_server.bat 启动    → 即时
第 5 步  打开浏览器验证               → 即时
第 6 步  下载 Qwen 模型 + 生成词典    → 约 1-2 小时（挂着就行）
```

## 第 1 步：安装 PostgreSQL

你说不确定是否装了，先检查一下：

```
打开 CMD 或 PowerShell，输入:
psql --version
```

如果显示版本号（如 `psql (PostgreSQL) 16.x`），说明已安装，跳到第 2 步。

### 没装的话，按以下步骤安装：

1. **下载 PostgreSQL 16**
   - 访问 https://www.enterprisedb.com/downloads/postgres-postgresql-downloads
   - 选择 **Windows x86-64** → 最新的 16.x 版本
   - 下载 `.exe` 安装包（约 300MB）

2. **安装过程**
   - 双击运行安装包
   - 安装目录：保持默认 `C:\Program Files\PostgreSQL\16`
   - 选择组件：全选（PostgreSQL Server、pgAdmin 4、Command Line Tools）
   - 数据目录：保持默认
   - **⚠️ 设置 postgres 超级用户密码：`123456`**（与项目 .env 配置一致）
   - 端口：保持默认 `5432`
   - 区域：保持默认
   - 点 Next 完成安装

3. **验证安装**
   - 安装完成后，打开新的 CMD 窗口
   - 如果 `psql --version` 仍然找不到命令，需要手动添加 PATH：
     ```
     系统设置 → 高级系统设置 → 环境变量 → Path → 新增:
     C:\Program Files\PostgreSQL\16\bin
     ```
   - 重新打开 CMD，再次验证

4. **确认 PostgreSQL 服务在运行**
   - 按 `Win+R` → 输入 `services.msc` → 回车
   - 找到 `postgresql-x64-16`，确认状态是 **"正在运行"**
   - 如果没有运行，右键 → 启动

## 第 3 步：一键初始化

### 方法 A：双击运行（推荐）

直接双击 `D:\wordbook_app\setup.bat`

它会自动完成：
1. 检测 Python 和 PostgreSQL
2. 创建 Python 虚拟环境
3. 安装所有 Python 依赖
4. 创建 `wordbook_db` 数据库
5. 执行建表 SQL（10 张表）
6. 插入 5 本内置词书
7. 创建管理员账号

### 方法 B：手动执行（如果 bat 脚本出问题）

打开 CMD，逐步执行：

```bash
# 进入项目目录
cd /d D:\wordbook_app\backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 初始化数据库（自动创建库 + 建表）
python -m scripts.init_db

# 创建管理员账号
python -m scripts.create_admin
```

### 常见问题排查

**❌ "连接被拒绝" / "Connection refused"**
→ PostgreSQL 服务没启动。打开 `services.msc`，找到 postgresql 服务，右键启动。

**❌ "密码认证失败" / "password authentication failed"**
→ PostgreSQL 密码不是 `123456`。两种解法：
  - 方法1：编辑 `backend\.env`，将 `DATABASE_URL` 中的 `123456` 改为你的实际密码
  - 方法2：重置 postgres 密码（用 pgAdmin 4 修改）

**❌ "pip install 很慢或超时"**
→ 使用国内镜像：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 第 6 步：生成词典数据

这一步用你的 RTX 5060 Ti 通过 Ollama 本地模型批量生成词典数据。

### 6.1 确保 Ollama 运行 + 模型已下载

```bash
# 启动 Ollama（如果还没启动）
ollama serve

# 另开一个窗口，下载 Qwen2.5-14B 模型（约 9GB，只需下载一次）
ollama pull qwen2.5:14b

# 验证模型可用
ollama list
```

### 6.2 批量生成 500 词词典数据

```bash
cd /d D:\wordbook_app\backend
venv\Scripts\activate

# 开始批量生成（约 1-2 小时，每个词 5-15 秒）
python -m scripts.batch_generate -i scripts/high_freq_500.txt -o scripts/generated_words.json
```

**可以随时中断！** 再次运行同一命令会自动跳过已生成的词（断点续跑）。

### 6.3 导入到数据库

```bash
# 将生成的数据导入到"高中英语词汇"词书
python -m scripts.import_to_db -i scripts/generated_words.json -w "高中英语词汇" --mark-reviewed
```

### 6.4 验证导入结果

浏览器打开 http://localhost:8000/docs

1. 用管理员账号登录拿 token
2. 调用 `GET /api/v1/wordbooks` → 看到"高中英语词汇"的 word_count 应该有数据了
3. 调用 `GET /api/v1/words/search?q=abandon` → 能搜到完整词典数据

## 后续迁移到阿里云

当本地验证完毕，迁移到云端只需：

1. 在阿里云创建 RDS PostgreSQL → 执行同一份 `001_init_schema.sql`
2. 修改 `.env` 中的 `DATABASE_URL` 指向阿里云 RDS
3. 后端部署到阿里云 ECS 或函数计算
4. Flutter Web 构建后上传到 OSS 做静态托管

架构不变，只是把数据库地址从 `localhost` 换成阿里云的地址。
